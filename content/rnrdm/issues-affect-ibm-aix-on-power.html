Bug 28745602

Creation of Grid Naming Service (GNS) client data manifest file from a standalone cluster using the command srvctl export gns -clientdata <manifest_xml_file> -role client fails with the following error:  

"java.lang.StackOverflowError: operating system stack overflow
error message:
./srvctl export gns -clientdata /scratch/gnswrap.xml -role client  Exception in thread "main" java.lang.StackOverflowError: operating system  stack overflow 

Workaround:

Update the srvctl command by adding -Xmso2048k to JAVA_STACK_SIZE   so that  JAVA_STACK_SIZE="-Xss2048k -Xmso2048k .

Bug 28390266

Running Cluster Verification Utility (CVU) resource causes memory consumption.

Workaround:

To reduce or prevent memory leak or memory consumption in CVU resource:

Install a patch in AIX systems to address a memory leak observed in the pthread library. IBM provides a list of Authorized Program Analysis Report (APAR) numbers for different versions.

The list of APAR numbers, AIX release or Technology Levels (TLs) are:

AIX Release/ TL Number APAR Number
AIX 6.1 TL09 IJ11198  
AIX 7.1 TL04  IJ11193
AIX 7.1 TL05 IJ11195
AIX 7.2 TL01 IJ11194
AIX 7.2 TL02 IJ11196
AIX 7.2 TL03 IJ11197

Bug 29053538

Starting with Oracle Database 18c, Oracle database instance at startup loads libora_netlib.so. This library has dependencies on IBM XL Fortran runtime libraries. If they are not installed on the system, startup fails with the following error: 

ORA-40238: invalid linear algebra shared library

Workaround:

Install XL Fortran Runtime for AIX Fix Pack 10 (February 2018 PTF) for 15.1.

Download and install XL Fortran Runtime for AIX Fix Pack 10 (February 2018 PTF) for 15.1 from:

https://www-01.ibm.com/support/docview.wss?uid=swg24044611
