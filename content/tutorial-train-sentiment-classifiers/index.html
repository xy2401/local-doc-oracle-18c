Training Sentiment Classifiers with Oracle Text

Before You Begin

This 5-minute tutorial shows you how to train a sentiment classifier so that classification rules provide a positive or negative sentiment about a search keyword.

Background

You train the sentiment classifier by using the set of sample documents and the specified preference and by assigning each sample document to a particular category. Then, Oracle Text deduces a set of classification rules that defines how to perform sentiment analysis.

What Do You Need?

  • Oracle Database 18c
  • Oracle SQL Developer 17.3.2

Create the Training Set

  1. Connect to myuser by using your myuser_conn connection. After the connection is established, a SQL worksheet opens and you use it to run SQL commands.
  2. Create the training table with review_id and text columns.
    3. CREATE TABLE training_camera(review_id NUMBER PRIMARY KEY, text VARCHAR2(2000));

    The training table contains the actual text of the training document set.

  3. Populate the training table with reviews.
    INSERT INTO training_camera VALUES(1,'This camera is OK!');
INSERT INTO training_camera VALUES(2,'The camera is absolutely fantastic!');
INSERT INTO training_camera VALUES(3,'The camera is terrible!');

Define the Categories

  1. Create the category table with doc_id, category, and category_desc columns.
    2. CREATE TABLE train_category(doc_id NUMBER, category NUMBER, category_desc VARCHAR2(100));

    The category table specifies training labels for the documents in the training table. The primary key of the training table must have a foreign key relationship with the unique key of the category table.

  2. Populate the category table with training labels.
    3. INSERT INTO train_category VALUES(2,1,'positive');
INSERT INTO train_category VALUES(3,2,'negative');

    Neutral categories are not supported, so you exclude neutral documents from the category table.

Train the Sentiment Classifier

  1. Create a CONTEXT index on the training set.
    2. CREATE INDEX docx ON training_camera(text) indextype IS 
CTXSYS.CONTEXT PARAMETERS('nopopulate');

    The CONTEXT index provides a framework to extract features from the training documents while training the sentiment classifier.

    A CONTEXT index with the NOPOPULATE argument creates the framework that is required for the training process to extract features from the document. It enables you to define different lexers or filters during the extraction. The NOPOPULATE argument also saves time as you don't use the actual index data.

  2. Train the sentiment classifier.
    3. EXEC CTX_CLS.SA_TRAIN_MODEL('clsfier','docx','review_id','train_category','doc_id','category','clsfier_camera')

    In the previous example, clsfier is the name of the sentiment classifier that is being trained, docx is the name of the index associated with the training set, review_id is the name of the document ID column in the document training set, train_category is the name of the category table that contains the labels for the training set documents, doc_id is the document ID column in the category table, category is the category column in the category table, and clsfier_camera is the name of the sentiment classifier preference that is used to train the classifier.

    After the sentiment classifier is trained, a set of classification rules are generated by Oracle Text. These classification rules define how to perform sentiment analysis by using the sentiment classifier.
