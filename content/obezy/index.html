Creating, Altering, and Setting Lockdown Profiles in Container Databases

Before You Begin

This 15-minute tutorial shows you how to create, alter, and set lockdown profiles at different levels in a container database (CDB), in the CDB root, in application roots, and application pluggable databases (PDB).

Background

You can set Oracle Database 12c lockdown profiles only in a CDB root, whereas you can set Oracle Database 18c lockdown profiles in application roots and application PDBs.

What Do You Need?

  • Familiarity with Oracle Database 12c lockdown profiles
  • Oracle Database 18c installed
  • A container database
  • A regular pluggable database
  • The hr_root.sh shell script and the create_hr_root.sql, create_hr_app.sql, create_operations.sql, create_sales.sql, and sync.sql SQL scripts downloaded to the /home/oracle/labs directory with passwords updated in those scripts.

Create Lockdown Profiles in the CDB and Application Roots

  1. Ensure that  tnsnames.ora is updated with entries for HR_ROOT, OPERATIONS, and SALES PDBs.
  2. Verify the value of ORACLE_HOME on your server. If necessary update the /home/oracle/labs/hr_root.sh shell script.
  3. Create the HR_ROOT application root and its two application PDBs (SALES and OPERATIONS). 
    /home/oracle/labs/hr_root.sh
  4. Log in to the CDB root. 
    sqlplus / AS SYSDBA
  5. Create the CDB_prof1 lockdown profile and its attributes. 
    CREATE LOCKDOWN PROFILE CDB_prof1;
ALTER LOCKDOWN PROFILE CDB_prof1 DISABLE STATEMENT = ('alter system');
ALTER LOCKDOWN PROFILE CDB_prof1 ENABLE STATEMENT = ('alter system')
                                 CLAUSE = ('set');
  6. Create the CDB_prof2 lockdown profile and its attributes. 
    CREATE LOCKDOWN PROFILE CDB_prof2;
ALTER LOCKDOWN PROFILE CDB_prof2 DISABLE STATEMENT = ('alter pluggable database');
  7. Verify that the lockdown profiles were created in the CDB root. 
    SELECT profile_name, rule, clause, status FROM cdb_lockdown_profiles
WHERE  profile_name LIKE 'CDB%';

PROFILE_NAME   RULE                     CLAUSE             STATUS
-------------- ------------------------ ------------------ -------
CDB_PROF1      ALTER SYSTEM                                DISABLE
CDB_PROF1      ALTER SYSTEM             SET                ENABLE
CDB_PROF2      ALTER PLUGGABLE DATABASE                    DISABLE
  8. Log in to the HR_ROOT application root as HR_LOCK_MGR. 
    CONNECT hr_lock_mgr@HR_ROOT 
Enter password: password
  9. Create the app_root_prof lockdown profile and its attributes. 
    CREATE LOCKDOWN PROFILE app_root_prof;
ALTER LOCKDOWN PROFILE app_root_prof DISABLE STATEMENT = ('alter system');
ALTER LOCKDOWN PROFILE app_root_prof ENABLE STATEMENT = ('alter system')
               CLAUSE = ('flush shared_pool');
  10. Verify that the lockdown profile was created in the application root. 
    SELECT profile_name, rule, clause, status FROM cdb_lockdown_profiles;
  
PROFILE_NAME   RULE                     CLAUSE             STATUS
-------------- ------------------------ ------------------ -------
APP_ROOT_PROF  ALTER SYSTEM                                DISABLE
APP_ROOT_PROF  ALTER SYSTEM             FLUSH SHARED_POOL  ENABLE
  11. Create a lockdown profile for the SALES application PDB. 
    CREATE LOCKDOWN PROFILE sales_prof;
ALTER LOCKDOWN PROFILE sales_prof                        
DISABLE STATEMENT = ('alter pluggable database');
  12. Verify that the lockdown profile was created in the application root. 
    SELECT profile_name, rule, clause, status FROM cdb_lockdown_profiles;

PROFILE_NAME   RULE                     CLAUSE             STATUS
-------------- ------------------------ ------------------ -------
APP_ROOT_PROF  ALTER SYSTEM                                DISABLE
APP_ROOT_PROF  ALTER SYSTEM             FLUSH SHARED_POOL  ENABLE
SALES_PROF     ALTER PLUGGABLE DATABASE                    DISABLE

Set a Lockdown Profile in the CDB Root

  1. Connect to the CDB root. 
    CONNECT / AS SYSDBA
  2. Check which lockdown profile is set in the CDB root. 
    SHOW PARAMETER pdb_lockdown
NAME                    TYPE        VALUE
----------------------- ----------- ------------------------------
pdb_lockdown            string
  3. Set the CDB_prof1 lockdown profile in the CDB root. 
    ALTER SYSTEM SET pdb_lockdown = CDB_prof1 SCOPE = both;
    SHOW PARAMETER pdb_lockdown
NAME                    TYPE        VALUE
----------------------- ----------- ------------------------------
pdb_lockdown            string      CDB_prof1

Set a Lockdown Profile in the Application Root

  1. Connect to the HR_ROOT application root as HR_LOCK_MGR. 
    CONNECT hr_lock_mgr@HR_ROOT
Enter password: password
  2. Check which lockdown profile is set in the application root. 
    SHOW PARAMETER pdb_lockdown
NAME                    TYPE        VALUE
----------------------- ----------- ------------------------------
pdb_lockdown            string      CDB_PROF1
  3. Set the CDB_prof1 lockdown profile in the application root. 
    ALTER SYSTEM SET pdb_lockdown = app_root_prof SCOPE = both;
    SHOW PARAMETER pdb_lockdown
NAME                    TYPE        VALUE
----------------------- ----------- ------------------------------
pdb_lockdown            string       APP_ROOT_PROF

Execute Allowed and Disallowed Commands in PDBs

  1. Connect to HR_ROOT as SYSTEM. 
    CONNECT system@HR_ROOT
Enter password: password
  2. Execute a command allowed by the application lockdown profile. The app_root_prof lockdown profile affects all application PDBs in the application container, but the application root still inherits the rules of its nearest ancestor, CDB_prof1. 
    ALTER SYSTEM SET ddl_lock_timeout=30 scope=both;
  3. Execute a command that is disallowed by the CDB root lockdown profile. 
    ALTER SYSTEM flush shared_pool;
ALTER SYSTEM flush shared_pool
*
ERROR at line 1:
ORA-01031: insufficient privileges
  4. Execute another command that is disallowed by the CDB root lockdown profile. 
    ALTER SYSTEM CHECKPOINT;
ALTER SYSTEM CHECKPOINT
*
ERROR at line 1:
ORA-01031: insufficient privileges
  5. Connect to SALES as SYS and display the inherited lockdown profile. 
    CONNECT sys@SALES AS SYSDBA
Enter password: password
    SHOW PARAMETER pdb_lockdown
----------------------- ----------- ------------------------------
NAME                    TYPE        VALUE
pdb_lockdown            string      APP_ROOT_PROF
  6. Execute the commands that are disallowed by the application root lockdown profile. 
    ALTER SYSTEM SET ddl_lock_timeout=30 scope=both;
ALTER SYSTEM SET ddl_lock_timeout=30 scope=both
*
ERROR at line 1:
ORA-01031: insufficient privileges
  7. Execute another command that is disallowed by the application lockdown profile whose restrictions are inherited from the CDB root profile. 
    ALTER SYSTEM flush shared_pool;
ALTER SYSTEM flush shared_pool
*
ERROR at line 1:
ORA-01031: insufficient privileges
  8. Execute another command that is disallowed by the application lockdown profile whose restrictions are inherited from the CDB root profile. 
    ALTER SYSTEM CHECKPOINT;
ALTER SYSTEM CHECKPOINT
*
ERROR at line 1:
ORA-01031: insufficient privileges

Set a Lockdown Profile in the Application PDB

  1. In the SALES application PDB, set the sales_prof lockdown profile. 
    ALTER SYSTEM SET pdb_lockdown = sales_prof SCOPE = both;
ALTER SYSTEM SET pdb_lockdown = sales_prof SCOPE = both
*
ERROR at line 1:
ORA-01031: insufficient privileges
  2. Connect to HR_ROOT. 
    CONNECT hr_lock_mgr@HR_ROOT
Enter password: password
  3. Unset the app_root_prof lockdown profile. 
    ALTER SYSTEM SET pdb_lockdown = '' SCOPE = both;
  4. Connect to SALES. 
    CONNECT hr_lock_mgr@SALES
Enter password: password
  5. Set the application lockdown profile at the application PDB level. 
    ALTER SYSTEM SET pdb_lockdown = sales_prof SCOPE = both;
  6. Connect to HR_ROOT. 
    CONNECT hr_lock_mgr@HR_ROOT
Enter password: password
  7. Reset the application lockdown profile in the application root. 
    ALTER SYSTEM SET pdb_lockdown = app_root_prof SCOPE = both;

Execute Allowed and Disallowed Commands in PDBs

  1. Connect to SALES. 
    CONNECT sys@SALES AS SYSDBA
Enter password: password
  2. Execute a command that is disallowed by the application PDB lockdown profile. 
    ALTER PLUGGABLE DATABASE sales CLOSE;
ALTER PLUGGABLE DATABASE sales CLOSE
*
ERROR at line 1:
ORA-01031: insufficient privileges
  3. Execute a command that is allowed by the CDB root lockdown profile. 
    ALTER SYSTEM SET ddl_lock_timeout=30 scope=both;
  4. Execute the commands that are disallowed by the CDB root lockdown profile. 
    ALTER SYSTEM flush shared_pool;
ALTER SYSTEM flush shared_pool
*
ERROR at line 1:
ORA-01031: insufficient privileges
    ALTER SYSTEM CHECKPOINT;
ALTER SYSTEM CHECKPOINT
*
ERROR at line 1:
ORA-01031: insufficient privileges

    Because a lockdown profile is set in the application PDB, the application PDB inherits the restrictions of the application PDB lockdown profile and the CDB root lockdown profile. It does not inherit restrictions from the application root lockdown profile.

Drop Lockdown Profiles

  1. Connect to the CDB root. 
    CONNECT / AS SYSDBA
Enter password: password
  2. Unset the CDB_prof1 lockdown profile. 
    ALTER SYSTEM SET pdb_lockdown = '' SCOPE = both;
  3. Drop the CDB_prof1 and CDB_prof2 lockdown profiles. 
    DROP LOCKDOWN PROFILE CDB_prof1;
    DROP LOCKDOWN PROFILE CDB_prof2;
  4. Connect to SALES. 
    CONNECT sys@SALES AS SYSDBA
Enter password: password
  5. Unset the sales_prof lockdown profile. 
    ALTER SYSTEM SET pdb_lockdown = '' SCOPE = both;
  6. Connect to HR_ROOT and drop the sales_prof lockdown profile. 
    CONNECT hr_lock_mgr@HR_ROOT 
Enter password: password
    DROP LOCKDOWN PROFILE sales_prof;
  7. To remove the ALTER SYSTEM SET restriction, update the existing app_root_prof lockdown profile. 
    ALTER LOCKDOWN PROFILE app_root_prof
               ENABLE STATEMENT = ('alter system') CLAUSE = ('set');
  8. Quit the session. 
    EXIT
