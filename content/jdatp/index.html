Autonomous Transaction Processing Database, SpringBoot and JDBC

Before You Begin

This tutorial shows you how to connect to an Autonomous Transaction Processing Database using JDBC and Spring.

Background

An Autonomous Transaction Processing Database enables convenience and ease of use. It allows autonomous backups and patching, zero downtime scaling, and easy provisioning.

The following OBE provides step by step instructions on how to connect to an Autonomous Transaction Processing Database using SpringBoot, which uses JDBC and UCP.

SpringBoot enables you to create standalone Spring based applications that are easy to set up and run.

Visit the SpringBoot website for more information: SpringBoot

This application accesses the EMP table in the database, and displays employee names in the table.

What Do You Need?

  • An integrated development environment such as Eclipse, Netbeans, IntelliJ, or JDeveloper.
  • Access to application code on Github.
  • An Oracle Cloud account (or access to an Autonomous Transaction Processing Database account).
  • Download the wallet files using your Cloud credentials (or, client credentials to connect to Autonomous Transaction Processing Database)
  • An EMP table in the database, populated with data. If needed, use script on Github to create and populate this table.

Download application code from Github.

Download and install Application and IDE

  1. Ensure that you have an IDE running. In this example, we are using Eclipse.

    2. Note: You may also use NetBeans, IntelliJ, or JDeveloper.

  2. Install the application code that you downloaded from Github.

Open the Java code in an IDE

  1. Download the application code from Github.
  2. Open a new project in Eclipse. Go to File. Click New, and then select Project.

Place the JAR files in the appropriate directory

  1. Ensure that the JDBC, Spring and commons-logging JAR files are present in the classpath/buildpath.

In this example, the JAR files are stored in the following location: D:\ATP_OBE\SpringBoot_JDBC_UCP_JARS\Spring_Boots_Jars\.


Update the URL and credentials

  1. Open the HelloAppConfig.xml file.
  2. Set the value of DB_URL to the destination on your local system, where you have downloaded the wallet.

    3. DB_URL="jdbc:oracle:thin:@jdbctest_medium?TNS_ADMIN=D:/ATP_OBE/wallet_JDBCTEST"

    Note: The wallet stores all information related to your database instance.

  3. Set the DB_USER to jdbcuser.
  4. Enter the DB password.

Test the code

  1. Once you have entered the credentials, go to Menu.
  2. Navigate to HelloApplication.java.
  3. Right click on the file. Select Run As. Then, select '1 Java Application'.

The code will compile and you will be able to view the output under the 'Console'.

Once the application is connected to the Autonomous Transaction Processing Database, you will be able to view the data in the table.
